# LLM 기반 우수사원 추천 AI 구현 계획

## 1. 기능 개요
- 근무 평가 데이터를 기반으로 GPT API 등 LLM을 활용해 분기별/부서별 우수사원을 자동 추천
- 추천 결과(직원명, 추천 사유 등)를 관리자에게 제공하여 포상/인정에 활용

## 2. 주요 단계

### Phase 1: 데이터 수집 및 전처리
- 평가 데이터(WorkEvaluation 엔티티)에서 필요한 항목 추출
- 부서/분기별로 집계 및 정규화(예: 점수 평균, 상위 N% 등)

### Phase 2: LLM API 연동
- OpenAI 등 GPT 기반 LLM API 연동
- 프롬프트 설계:  
  예시) "아래 직원 평가 데이터를 참고해 우수사원을 추천해줘. 추천 사유도 함께 작성해줘."
- 평가 데이터와 함께 API 요청, 추천 결과 파싱

### Phase 3: 추천 결과 제공 API 구현
- `/evaluations/recommend-reward` 엔드포인트 설계 (POST)
- 요청: { year, quarter, departmentId }
- 응답: [{ employeeId, name, reason }]
- 관리자 화면에서 추천 결과 조회 및 포상 등록 가능

### Phase 4: 테스트 및 품질 관리
- 다양한 평가 데이터로 추천 결과 검증
- 추천 결과의 신뢰성 및 다양성 테스트
- 예외 처리 및 API 응답 안정성 확보

### Phase 5: 운영 및 모니터링
- 추천 결과 로그 및 피드백 수집
- LLM API 사용량/비용 모니터링
- 추천 결과 품질 개선을 위한 프롬프트/파라미터 튜닝

## 3. 기술 스택 및 의존성
- Spring Boot (REST API)
- OpenAI GPT API (또는 Azure OpenAI, KoGPT 등)
- JPA/Hibernate (평가 데이터 관리)
- Lombok, Swagger (문서화 및 개발 생산성)
- (옵션) 캐싱, 비동기 처리, 배치 추천 등

## 4. 예시 API 흐름
1. 관리자 → 추천 요청 (프론트)
2. 백엔드 → 평가 데이터 집계 → GPT API 호출
3. GPT → 추천 결과 반환 (추천 직원, 사유)
4. 백엔드 → 프론트에 결과 반환 → 관리자 화면 표시

## 5. 향후 확장 계획
- 추천 기준 커스터마이징(가중치, 부서별 특성 등)
- 추천 결과에 대한 관리자 피드백 반영
- 포상 이력 관리 및 통계 기능 추가
